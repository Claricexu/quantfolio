# Quantfolio — Developer Guide

Handoff documentation for developers taking over this project. For end-user instructions, see [USER_GUIDE.md](USER_GUIDE.md). For project-level overview, see [README.md](README.md).

This guide assumes you already know Python, pandas, scikit-learn, and have read the README architecture diagrams.

---

## 1. Environment setup

- **Python 3.11+** (tested on 3.11)
- **Platform:** Windows 10/11 is the primary development target. Linux/macOS should work but are untested end-to-end.
- **Conda strongly recommended** on Windows. [`start_dashboard.bat`](start_dashboard.bat) auto-activates `%USERPROFILE%\miniconda3`. If you use a different env manager, you'll need to edit the batch file or launch with `python api_server.py` directly.
- **Install:** `pip install -r requirements.txt`
- **Configure secrets:** copy `.env.example` → `.env`, fill in values. See [README §Quick Start #2](README.md#2-configure-env-optional-but-recommended).

### Windows-specific quirks to know about

| Quirk | Why |
|---|---|
| All RF/XGB models use `n_jobs=1` | Avoids joblib deadlock on Windows under the FastAPI threadpool. Don't "optimize" by setting `n_jobs=-1` without testing under load. |
| [`start_dashboard.bat`](start_dashboard.bat) checks for `fastapi` before running pip | So users don't re-install on every launch. If you add a new dependency, first-time users will still get it, but existing users may need to run `pip install -r requirements.txt` manually. |
| SQLite DB (`fundamentals.db`) is ~765 MB | Gitignored — regenerated by `edgar_fetcher.py`. Cold rebuild takes ~3.5 hours because SEC EDGAR rate-limits at 10 req/sec. |

## 3. Architecture in 60 seconds

- **Layer 2 (stable, do not touch casually):** FastAPI backend → runs Lite/Pro models on a universe of 174 symbols → serves predictions + backtests to the dashboard. See [README.md](README.md) for the full tour.
- **Layer 1 (pipeline):** SEC EDGAR → ~2,500 tickers → prescreen to ~1,414 → fundamentals screen by archetype → top 100 → `leaders.csv`. See [good_firm_framework.md](good_firm_framework.md) for the rubric and [TWO_LAYER_ARCHITECTURE_PLAN.md](TWO_LAYER_ARCHITECTURE_PLAN.md) for the decision history.
- **The bridge:** [`get_all_symbols()`](finance_model_v2.py:146) in `finance_model_v2.py` reads `leaders.csv ∪ Tickers.csv` and exposes the 174-symbol universe to Layer 2.

## 4. Extension recipes

### 4.1 Add a new symbol manually
Edit [Tickers.csv](Tickers.csv), add a line, restart the server. First lookup/backtest downloads price history from Yahoo.

### 4.2 Add an ETF (so Full Signal strategy triggers automatically)
Add the ticker to `ETF_TICKERS` in [`finance_model_v2.py:82`](finance_model_v2.py:82).

### 4.3 Add a new Good Firm test
- Decide archetype: GROWTH or MATURE
- Write `def _test_<archetype>_<name>(m)` following the pattern in [`fundamental_screener.py:56-188`](fundamental_screener.py:56) — it takes a metrics dict `m`, returns `True`/`False`/`None` (None = insufficient data).
- Add it to [`MATURE_TESTS`](fundamental_screener.py:112) or [`GROWTH_TESTS`](fundamental_screener.py:188) as a `(name, fn)` tuple.
- If the test needs a new metric, add it in [`fundamental_metrics.py`](fundamental_metrics.py) first.
- Re-run `python fundamental_screener.py --all` to regenerate `screener_results.csv`.
- **Important:** anchor-test your change. Known-good anchors are KO/JNJ/PG/WMT/MCD (must stay MATURE LEADER/GEM) and NVDA/MSFT/META/CRWD/NOW (must stay GROWTH LEADER/GEM). See `diagnostics/diag_anchor_breakdown.py` for the reference checker.

### 4.4 Add a new ML model
The cleanest pattern is "v4" alongside existing v2/v3:
- Add `engineer_features_v4()`, `train_*_v4()`, `predict_v4()` in [`finance_model_v2.py`](finance_model_v2.py) (or a new `finance_model_v4.py`).
- Route it through the `version` parameter in `predict_ticker()` and downstream.
- Add the new version to `MODEL_VERSION` auto-selection at [`finance_model_v2.py:74`](finance_model_v2.py:74) if it should become the default.
- Backtest against SPY/QQQ/MU/META/SMH and compare Sharpe before shipping.

### 4.5 Add a new API endpoint
Add a FastAPI route in [`api_server.py`](api_server.py). The pattern is:
```python
@app.get("/api/your-endpoint/{param}")
def your_endpoint(param: str):
    ...
    return {"data": ...}
```
The frontend (`frontend/index.html`) calls endpoints via `fetch()` directly — no separate SDK.

### 4.6 Add a new dashboard tab
The frontend is a single-file SPA in [`frontend/index.html`](frontend/index.html). Add:
1. A `<button>` tab trigger in the nav
2. A `<div class="tab-pane">` for the content
3. JavaScript to render data from your new endpoint

No build step, no framework — just vanilla JS + Chart.js.

### 4.7 Change the prescreen thresholds
Edit [`prescreen_rules.json`](prescreen_rules.json) (market cap floor, price floor, volume floor). Re-run `python universe_builder.py --prescreen-only` to regenerate without re-hitting SEC.

## 5. Scheduled jobs

APScheduler is configured in [`api_server.py:317-340`](api_server.py:317). Two cron jobs are registered on server start:

| Schedule | What it does |
|---|---|
| **4:05 PM EST, Mon–Fri** | Runs the dual-model scan across all 174 symbols, caches the result, fires email alert if any HIGH-confidence signals |
| **Feb 15 / May 15 / Aug 15 / Nov 15 @ 2 AM** | Triggers the Layer 1 quarterly rebuild (universe_builder → edgar_fetcher → fundamental_screener → leader_selector) |

Both run in the in-process `BackgroundScheduler`. If APScheduler isn't installed, scheduling is silently skipped — the endpoints still work manually via `?refresh=true`.

## 6. Caching layers

| Cache | TTL | Location | Invalidated by |
|---|---|---|---|
| Price CSVs | Per trading day | `data_cache/*.csv` | Delete the file to force re-fetch |
| Daily report | 22 hours | In-memory + `data_cache/report_cache.json` | `?refresh=true` on `/api/report` |
| Backtest JSONs | 7 days | `data_cache/backtest_*.json` | `?refresh=true` on `/api/backtest-chart/*` |
| SEC XBRL facts | 90 days | `fundamentals.db` | `edgar_fetcher.py --refresh` |
| SEC ticker list | Per rebuild | `universe_raw.csv` | `universe_builder.py --build` |

`data_cache/` is gitignored and auto-created at launch.

## 7. External APIs & rate limits

| API | Used by | Rate limit | Auth |
|---|---|---|---|
| **Yahoo Finance** (yfinance) | [`finance_model_v2.py`](finance_model_v2.py), pricing + `.info` for SVR | Informal — ~2 req/sec is safe; 1-minute cool-downs on burst | None, but IP-based rate limits can bite |
| **SEC EDGAR** | [`edgar_fetcher.py`](edgar_fetcher.py), [`universe_builder.py`](universe_builder.py), diagnostics | **Hard: 10 req/sec**. Cold universe fetch ≈ 3.5 hrs | `User-Agent` header required with a real contact email (see `SEC_USER_AGENT` in `.env`) |

**If SEC starts returning 403s**, the User-Agent is either missing or non-identifying. Check [`edgar_fetcher.py:37`](edgar_fetcher.py:37) and verify `.env` is loading.

## 8. Testing

**Known gap: there is no automated test suite.** Model correctness has been verified empirically via backtests (see README §Backtest Results), but there are no unit tests or integration tests.

If you're adding tests, highest-leverage first targets:

1. **`tests/test_screener_verdicts.py`** — anchor-test that KO/JNJ/PG/WMT/MCD land MATURE LEADER/GEM and NVDA/MSFT/META/CRWD/NOW land GROWTH LEADER/GEM. This is the single most important regression barrier.
2. **`tests/test_env_config.py`** — verify `.env` loading doesn't silently disable SMTP.
3. **`tests/test_feature_engineering.py`** — snapshot test that `engineer_features_v2()` and `engineer_features_v3()` produce the expected feature count (13 and 22) and no NaN leakage at the edges.
4. **`tests/test_cache_ttl.py`** — verify the 22-hour report cache and 7-day backtest cache behave correctly across the TTL boundary.

Recommended stack: `pytest` + `pytest-mock` for the yfinance and SEC HTTP calls.

## 9. Frontend

Single file: [`frontend/index.html`](frontend/index.html). ~116 KB. Vanilla JS + inline CSS + Chart.js from CDN. No build, no bundler, no framework.

This is intentional — the whole app is "launch a batch file, open localhost." Adding a build step would break that promise. If the frontend grows past ~2000 LOC, revisit.

## 10. Diagnostics folder

[`diagnostics/`](diagnostics) contains 18 one-off scripts, each tied to a specific Phase 1.x decision. Most are **archival** — they were run once to resolve a question, the decision got locked into the rubric in [`good_firm_framework.md`](good_firm_framework.md), and the script is kept for provenance.

**Live / still-useful:**
- `diag_threshold_sensitivity.py` — re-run if you change the 12% GROWTH/MATURE cut
- `diag_rubric_audit.py` — full verdict breakdown across the universe
- `diag_anchor_breakdown.py` — anchor-ticker regression check
- `diag_archetype_split.py` — counts GROWTH vs MATURE across sectors

**Archival** (ran once, don't need to re-run):
- `diag_bdc_sic.py`, `diag_cat_a_tags*.py`, `diag_crwd_sec_tags.py`, `diag_blank_sic.py`, `diag_dual_class_and_googl.py`, `diag_lly_opmargin.py`, `diag_unknown_triage.py`, `diag_track_c.py`, `diag_db_coverage.py`

When writing a new diagnostic, follow the `diag_<topic>.py` convention and document the outcome in a comment block at the top (see existing examples).

## 11. Known limits & Phase 2 backlog

See the "Known Unfixable & Phase 2 Backlog" section in [good_firm_framework.md](good_firm_framework.md). Highlights:

- **Cyclicals** (energy, auto, retail) trip MATURE's `cagr_shrinking` dealbreaker during normal cycle troughs → need a CYCLICAL sub-archetype
- **Regulated utilities** fail `cash_generation` because capex suppresses FCF margin → need a UTILITY sub-archetype
- **CostOfServices gap** — ~10% of $1B+ service-dominant tickers have null `gross_margin_ttm` from XBRL
- **`dividend_yield` unit inconsistency** from yfinance — dormant, would block future dividend-based refinement

## 12. Common pitfalls

| Symptom | Likely cause |
|---|---|
| `start_dashboard.bat` opens then immediately closes | Conda activation failed. Open Anaconda Prompt manually, `cd` to folder, run `python api_server.py` to see the real error. |
| Email alerts silently stop firing | `.env` missing or `SMTP_ENABLED=false`. Check the Command Prompt console — it prints `[Alert] ...` lines when SMTP is wired up. |
| Leader Detector rebuild hangs at "Fetching XBRL" | Normal — SEC's 10 req/sec floor makes cold rebuilds take ~3.5 hrs. Don't cancel; the fetch is resume-safe (cached rows skip). |
| All models predict 0% change | Likely NaN leakage in feature engineering. Check the last 5 rows of the feature DataFrame — `.dropna()` should not eat today's row. |
| `RandomForestRegressor` hangs on Windows | `n_jobs` is not 1 somewhere. Search for `n_jobs=` and confirm every instance is `n_jobs=1`. |
| Batch backtest shows "cached" but results are stale | TTL on backtest JSONs is 7 days. Force-refresh via `?refresh=true` on `/api/backtest-chart/{symbol}`. |
